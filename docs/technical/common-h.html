    Macro Documentation

    DEFAULT_COMPARISON_CRITERION

    The DEFAULT_COMPARISON_CRITERION defines default comparison criterion used to decide whether a file in the source location is superior to its previously backed up version when performing backup synchronization. Initial value is CRITERION_MORE_RECENT which means that the most recent version of the file will be backed up.

    See also Criterion.

    DEFAULT_COPY_BUFFER_SIZE

    The DEFAULT_COPY_BUFFER_SIZE defines default size (in megabytes) of copy buffer used for file copy operations during the backup process. Initial value is 1 megabyte.

    DEFAULT_KEEP_OBSOLETE

    The DEFAULT_KEEP_OBSOLETE defines default boolean value which decides whether the application should keep obsolete files in the location of previously created backup after synchronization. Initial value is false.

    DEFAULT_LOCALE_CODE

    The DEFAULT_LOCALE_CODE defines default code of the language to display the application's user interface in. Initial value is sk which corresponds to Slovak (Slovakia).

    DEFAULT_LOGS_LOCATION

    The DEFAULT_LOGS_LOCATION macro holds the path to the default location where application's log files should be saved.

    DEFAULT_SYNCHRONIZE

    The DEFAULT_SYNCHRONIZE defines default boolean value which decides whether the application should only synchronize with a previously created backup. Initial value is false.

    GIGABYTE

    The GIGABYTE macro holds the number of bytes corresponding to one gigabyte.

    KILOBYTE

    The KILOBYTE macro holds the number of bytes corresponding to one kilobyte.

    LOCALE_RELATIVE_PATH

    The LOCALE_RELATIVE_PATH holds the path to the directory containing translation files for the application relatively to the location of application's executable.

    MEGABYTE

    The MEGABYTE macro holds the number of bytes corresponding to one megabyte.

    TERABYTE

    The TERABYTE macro holds the number of bytes corresponding to one terabyte.

    WORKER_ERROR_LOG_FILE_NAME

    The WORKER_ERROR_LOG_FILE_NAME defines the application's event log file name.

    WORKER_EVENT_LOG_FILE_NAME

    The WORKER_EVENT_LOG_FILE_NAME defines the application's event log file name.
